# Лабораторная работа №3_1
## Тема: Асинхронное взаимодействие микросервисов с RabbitMQ

### Цель работы
Реализовать синхронное взаимодействие через gRPC и асинхронное через RabbitMQ с развертыванием в Docker.

### Стек технологий
- Ubuntu 20.04, Python 3.10
- grpcio, grpcio-tools, pika
- Docker, RabbitMQ

### Вариант 13

| № | Задание |
|---|---------|
| 1 | Сброс пароля: по email сгенерировать токен и вернуть `"Ссылка для сброса отправлена на {email}"` |
| 2 | Расчет возраста: по дате `YYYY-MM-DD` вернуть количество лет |
| 3 | Сортировка слов: из строки вернуть слова в алфавитном порядке |

### Архитектура

**Часть 1 (gRPC):** Клиент → напрямую → gRPC сервер (порт 50051)

**Часть 2 (RabbitMQ + gRPC):** Producer → очередь RabbitMQ → Consumer → gRPC сервер

### docker-compose.yml
```yaml
version: '2.2'
services:
  rabbitmq:
    image: rabbitmq:3.9-management
    container_name: rabbitmq
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: user
      RABBITMQ_DEFAULT_PASS: password
```

## Запуск системы
bash
# Терминал 1: gRPC сервер

```cd grpc_sync && python3 grpc_server.py```

# Терминал 2: RabbitMQ

```cd rabbitmq_async && docker-compose up -d```

# Терминал 3: Consumer

```cd rabbitmq_async && python3 consumer.py```

# Терминал 4: Producer
```python3 producer.py email:test@mail.ru

python3 producer.py birth:2000-05-12

python3 producer.py sort:яблоко банан апельсин
```
