# Лабораторная работа №1

## Вариант 13

### Тема: gRPC-сервис EmailService (Unary RPC)

---

## Задание

Реализовать gRPC-сервис EmailService с методом:

SendEmail(EmailMessage)

Метод должен принимать одно сообщение с данными письма и возвращать один ответ  
(Unary RPC).

Клиент вводит данные письма вручную и отправляет их на сервер, после чего сервер обрабатывает данные и возвращает результат.

---

## Архитектура решения

В работе реализована клиент-серверная архитектура.

flowchart LR

Client[Client<br>client.py]

Server[gRPC Server<br>server.py]

Proto[Protocol Buffers<br>email.proto]

Generated[Generated Code<br>email_pb2.py<br>email_pb2_grpc.py]

Client -->|EmailMessage| Server
Server -->|EmailResponse| Client

Proto --> Generated

Generated --> Server
Generated --> Client

Компоненты системы

Client (client.py)  
Отправляет запрос с данными письма на сервер.

Server (server.py)  
Принимает запрос клиента, обрабатывает его и возвращает ответ.

email.proto  
Контракт взаимодействия между клиентом и сервером.  
Описывает сервис и структуры сообщений.

---

## Описание реализации

В рамках лабораторной работы был создан gRPC-сервис EmailService.

Описание сервиса выполнено в файле email.proto.

Сервис содержит метод:

- SendEmail — принимает одно сообщение с данными письма.

Метод не использует потоковую передачу данных, так как реализует Unary RPC. Клиент отправляет один запрос, а сервер возвращает один ответ.

Таким образом реализуется механизм Unary RPC.

[Cкриншот email.proto](https://raw.github.com/MakhoninaAV/-./main/5359542285860803541.jpg)

## Генерация gRPC-кода

После создания файла email.proto была выполнена команда:

python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. email.proto

В результате были автоматически созданы файлы:

- email_pb2.py  
- email_pb2_grpc.py  

Эти файлы используются для взаимодействия клиента и сервера через gRPC.

[Cкриншот подключения к серверу](https://raw.github.com/MakhoninaAV/-./main/5359542285860803544.jpg)

## Реализация сервера

Серверная часть реализована в файле server.py.

Создан класс EmailService, который наследуется от EmailServiceServicer.

В данном классе реализован метод:

SendEmail — принимает одно сообщение EmailMessage.

Сервер получает данные письма (получатель, тема, текст), выводит их в консоль и возвращает ответ с подтверждением отправки.

Сервер запускается на порту:

50051

и ожидает подключения клиентов.

(вставить скриншот запущенного сервера)

---

## Реализация клиента

Клиентская часть реализована в файле client.py.

Подключение к серверу выполняется через:

grpc.insecure_channel('localhost:50051')

Клиент вручную вводит:

- получателя письма  
- тему письма  
- текст письма  

После этого данные отправляются на сервер одним сообщением.

Сервер обрабатывает запрос и возвращает ответ, который выводится в консоль клиента.
